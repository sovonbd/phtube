Discuss the scope of var, let, and const

In JavaScript, the scope of variables declared using var, let, and const differs in terms of how they are accessible and where they "live" within the code. Understanding these scopes is crucial for writing clean and bug-free JavaScript code. Here's a breakdown of the scope of each:

var

  • Variables declared with var are function-scoped, meaning they are accessible within the function where they are defined.
  • 'var' variables are hoisted to the top of their containing function or global scope, but they are initialized with undefined.

let

  • Variables declared with let are block-scoped, meaning they are accessible only within the block where they are defined.
  • 'let' variables are also hoisted to the top of their containing block but are not initialized until the actual declaration is encountered in the code.

const

  • Like 'let', variables declared with const are block-scoped.
  • Variables declared with const cannot be reassigned once they are initialized. However, the value they hold (e.g., an object or array) can be modified.
Tell us the use cases of null and undefined

Use cases of null and undefined in JavaScript:

null

  • Explicitly set to indicate the absence of a value or an empty reference.
  • Often used when you want to represent that a variable or object property has no meaningful value or that it has been intentionally cleared.

undefined

  • Automatically assigned to variables or object properties that have been declared but not initialized.
  • Represents the absence of a value due to oversight or when a function doesn't return a value explicitly.
  • It's a sentinel value in JavaScript and should generally be avoided when possible to prevent unexpected behavior.
What do you mean by REST API?

REST API stands for Representational State Transfer Application Programming Interface. It is a standardized architectural style for designing networked applications. REST APIs use HTTP requests to perform CRUD (Create, Read, Update, Delete) operations on resources, which are represented as URLs. They are known for their simplicity, scalability, and statelessness, making them widely used for building web services and APIs.
